苍穹外卖-用户端接口文档

一、用户管理接口

1.1 用户注册

POST /user/users/register

请求参数 (JSON)
{
  "userName": "String",       // 用户名 (必填,唯一)
  "phone": "String",          // 手机号 (必填)
  "sex": "String",            // 性别 (必填)
  "idNumber": "String",       // 身份证号 (可选)
  "avatar": "String",         // 头像URL (可选)
  "openId": "String"          // 微信OpenId (必填)
}
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "/register::用户ID"
}

1.2 用户登录

POST /user/users/login

请求参数 (JSON)
{
  "userName": "String",        // 用户名 (必填)
  "password": "String"         // 密码 (必填)
}
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "用户ID::jwt_token字符串"
}

1.3 用户登出

POST /user/users/logout

请求头

需要携带JWT Token (UserToken)

响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "logout::用户ID"
}

五、购物车接口

5.1 添加购物车

POST /user/shoppingCarts

请求头

需要携带JWT Token (UserToken)

请求参数 (JSON)
{
  "dishId": "Long",           // 菜品ID (与setmealId二选一)
  "setmealId": "Long",        // 套餐ID (与dishId二选一)
  "dishFlavor": "String",     // 口味 (可选)
  "number": "Long"            // 数量 (必填,默认1)
}
说明

如果添加的是菜品,dishId必填;如果添加的是套餐,setmealId必填

七、订单接口

7.1 提交订单

POST /user/orders/submit

请求头

需要携带JWT Token (UserToken)

请求参数 (JSON)
{
  "addressBookId": "Long",         // 地址簿ID (必填)
  "payMethod": "Long",             // 支付方式: 1支付宝 2微信 (必填)
  "remark": "String",              // 备注 (可选)
  "orderDetails": [{               // 订单详情列表 (必填)
    "dishId": "Long",              // 菜品ID (与setmealId二选一)
    "setmealId": "Long",           // 套餐ID (与dishId二选一)
    "name": "String",              // 名称 (必填)
    "dishFlavor": "String",        // 口味 (可选)
    "number": "Long",              // 数量 (必填)
    "amount": "BigDecimal",        // 金额 (必填)
    "image": "String",             // 图片 (必填)
    "packAmount": "BigDecimal",    // 打包费 (可选)
    "tablewareNumber": "Long"      // 餐具数量 (可选)
  }]
}
响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": "submitOrder::订单ID"
}

八、店铺状态接口

8.1 获取店铺状态

GET /user/shop/status

响应示例
{
  "code": 200,
  "msg": "操作成功",
  "data": 1
}

说明: 返回1表示营业中，返回0表示已打烊

九、枚举说明

订单状态 (OrderStatus)

值说明
1待付款
2待接单
3已接单
4派送中
5已完成
6已取消
7退款

支付状态 (PayStatusEnum)

值说明
0未支付
1已支付
2退款

支付方式 (payMethod)

值说明
1支付宝
2微信

性别 (sex)

值说明
0女
1男

默认地址 (isDefault)

值说明
0非默认
1默认地址

分类类型 (type)

值说明
1菜品分类
2套餐分类

十、统一响应格式

{
  "code": 200,           // 200表示成功，其他表示失败
  "msg": "操作成功",    // 提示信息
  "data": {}           // 返回数据
}

十一、认证说明

以下接口需要携带JWT Token进行身份认证:

  • 购物车相关接口
  • 地址簿相关接口
  • 订单相关接口
  • 用户登出接口

请求头格式: Authorization: Bearer {token}
